# Homework 2 表單元素說明

本文件說明 `index.html` 中各表單欄位與 HTML 標籤的對應關係。

## 表單欄位對照表

| 介面欄位 | HTML 標籤  | Type 屬性 | 功能說明                           |
| :------- | :--------- | :-------- | :--------------------------------- |
| **姓名** | `<input>`  | text      | 基本的單行文字輸入框               |
| **帳號** | `<input>`  | text      | 單行文字輸入框，限制純英數字輸入   |
| **密碼** | `<input>`  | password  | 將輸入內容隱藏的密碼專用框         |
| **國家** | `<select>` | 無        | 提供多個選項的下拉式選單           |
| **興趣** | `<input>`  | checkbox  | 支援多重選擇的勾選框（複選）       |
| **性別** | `<input>`  | radio     | 同群組內只能擇一的單選按鈕         |
| **出生日期**| `<input>`| date      | 支援月曆介面的日期選擇器           |
| **年齡** | `<input>`  | number    | 限制只能輸入數字的數字輸入框       |
| **留言** | `<textarea>`| 無        | 支援多行輸入的文字區域             |
| **提交表單**| `<button>` | submit    | 點擊後可送出表單資料的按鈕         |

## 調整紀錄與後續方向

為了提升表單體驗與確保資料完整性，我們進行了前端驗證的升級：

### 實作流程與新增功能：
1. **攔截表單預設行為**：
   原先點擊提交會導致頁面跳轉至 404 找不到網頁。透過 JavaScript (`event.preventDefault()`) 成功取消此預設跳轉，使驗證動作能於當前頁面順利獨立執行。
2. **必填欄位與視覺提示**：
   重新梳理欄位需求，除了「留言」欄位允許空白外，其餘欄位皆改為必填。針對必填標籤額外加入紅色的星號（`*`）以便使用者一眼辨識。
3. **前端即時驗證與錯誤回饋**：
   - 使用者點擊「提交表單」時，系統會掃描所有必填項（包含文字輸入、下拉選單、單選/複選框）。
   - **未填寫齊全**：跳出「尚有資料未填寫齊全」的警告視窗，在每一個漏填的欄位下方即時顯示紅字「此為必要欄位」。
   - **驗證通過**：跳出「已經儲存」的成功訊息。此流程皆於前端把關，不再需要額外開啟 json 儲存。

### 未來可能調整方向：
- **更進階的欄位驗證**：目前主要確保「有填寫」，未來可加入格式檢查（如信箱格式正確性檢查、密碼強度長度限制、年齡與生日必須相符的邏輯等）。
- **驗證視覺優化**：加入輸入框的紅色邊框變化、加入錯誤訊息淡入淡出動畫，或是在使用者邊打字時及時清除該欄位的紅字錯誤提示。
- **正式串接後端**：待未來有後端環境與 API，即可將前台已經把關過的乾淨資料打包成 JSON，再透過 `fetch` 或 AJAX 等技術真實驗證並存入資料庫中。
